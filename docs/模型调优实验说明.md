# 煤炭配选大模型调优实验说明

## 1. 实验定位

本实验面向煤矿智能配煤系统中的“候选方案生成 + 方案解释生成”任务进行轻量领域适配。当前系统中，大模型已经参与候选配比方案生成，后端继续负责物料合法性校验、库存校验、质量预测和多目标评分排序。

这种定位与现有系统实现一致：大模型不是直接绕过后端给出最终方案，而是提出候选配比，系统再进行可行性与评分校验。该方式能够支撑任务书中“基于开源大模型煤炭配选垂直模型，输出可行配煤方案”的要求。

## 2. 基座模型

建议选择开源中文指令模型作为基座模型，例如：

- Qwen/Qwen2.5-1.5B-Instruct
- Qwen/Qwen2.5-3B-Instruct
- Qwen/Qwen2.5-7B-Instruct

本科毕设阶段优先选择 1.5B 或 3B，便于快速完成训练、评估和复现。

## 3. 数据来源

训练样本由主系统数据库快照和公开煤质资料共同构成，主要包含：

- 销售订单数据：需求量、灰分、硫分、水分、发热量、优先级、交付日期；
- 配煤方案数据：配比、预测煤质、成本、质量评分、成本评分、稳定性评分、综合评分；
- 规则知识数据：质量约束、库存约束、配比约束、经验规则；
- 历史案例数据：典型订单、历史配煤描述、执行效果；
- RAG 知识数据：规则、案例、术语和业务文档片段；
- 库存数据：煤种、仓库、可用量、批次信息。
- 公开煤质样本：水分、灰分、硫分、挥发分、发热量和来源说明。

数据构建命令：

```bash
python3 scripts/build_dataset.py
```

当前脚本会生成两类任务样本：

| 任务 | 输出格式 | 用途 |
|---|---|---|
| candidate_generation | `{"plans":[...]}` | 训练模型根据订单和候选物料生成候选配比 |
| plan_explanation | `{"ruleBasis":...}` | 训练模型对最终推荐方案生成结构化解释 |

样本规模以实际运行 `scripts/build_dataset.py` 的输出为准。

## 4. 调优目标

候选方案生成任务输入为订单、候选物料、规则、案例和 RAG 知识，输出必须为合法 JSON：

```json
{
  "plans": [
    {
      "planName": "低硫高热值候选方案A",
      "strategy": "优先满足低硫和热值约束，同时控制高价煤占比",
      "items": [
        {
          "coalId": 1,
          "productBatchNo": "PBxxx",
          "ratio": 0.6,
          "reason": "低硫、库存充足，作为主配物料"
        }
      ],
      "risk": "执行前需复核库存余量和煤质波动"
    }
  ]
}
```

方案解释任务输入为订单和推荐方案的完整上下文，输出必须为合法 JSON：

```json
{
  "ruleBasis": "规则依据",
  "caseReference": "案例参考",
  "recommendReason": "推荐理由",
  "riskTip": "风险提示",
  "finalExplanation": "最终解释"
}
```

调优目标包括：

1. 提高候选方案 JSON 输出格式稳定性；
2. 提高候选物料合法率，减少编造煤种、批次和指标；
3. 提高配比合法率，使每个方案 ratio 之和接近 1；
4. 提高后端可接收率和可行方案比例；
5. 提高规则依据与输入规则的一致性；
6. 提高风险提示与质量、库存、成本约束的相关性。

## 5. LoRA 训练方法

本实验采用 LoRA 参数高效微调。基座模型参数保持冻结，仅训练注意力层和前馈层中的低秩适配参数。建议初始参数如下：

| 参数 | 建议值 |
|---|---:|
| LoRA rank | 8 |
| LoRA alpha | 16 |
| LoRA dropout | 0.05 |
| epoch | 3 |
| learning rate | 2e-4 |
| max seq length | 2048 |

训练命令：

```bash
python3 scripts/train_lora.py --config configs/qwen_lora.yaml
```

## 6. 评估方法

使用验证集对基座模型和 LoRA 模型分别生成输出，统计以下指标：

| 指标 | 含义 |
|---|---|
| valid_json_rate | 模型输出能够被解析为合法 JSON 的比例 |
| complete_field_rate | 输出包含五个必需字段的比例 |
| nonempty_field_rate | 五个字段均非空的比例 |
| field_nonempty_rate | 各字段分别非空的比例 |
| valid_candidate_plan_rate | 候选方案字段、物料和配比均合法的比例 |
| valid_ratio_plan_rate | 候选方案 ratio 之和接近 1 的比例 |
| valid_item_count_plan_rate | 每个候选方案包含 2 到 4 种物料的比例 |

评估命令示例：

```bash
python3 scripts/evaluate_json_outputs.py \
  --base-model Qwen/Qwen2.5-1.5B-Instruct \
  --limit 20 \
  --report-file outputs/reports/base_eval_report.json

python3 scripts/evaluate_json_outputs.py \
  --base-model Qwen/Qwen2.5-1.5B-Instruct \
  --adapter outputs/adapters/qwen-coal-lora \
  --limit 20 \
  --report-file outputs/reports/lora_eval_report.json
```

## 7. 论文表述建议

建议在论文中这样表述：

> 本文基于开源指令大模型构建煤炭配选领域适配模型。结合系统数据库中的订单、配煤方案、规则知识、历史案例和 RAG 知识片段，构造指令微调样本，并采用 LoRA 参数高效微调方法对模型进行领域化适配。微调后的模型不直接替代后端配煤优化算法，而是用于提升方案解释、规则依据生成、案例引用和风险提示的稳定性。系统最终形成“多目标评分寻优 + RAG 知识增强 + 领域适配大模型解释生成”的协同决策流程。
